Silence Protocol (SIL)

SIL is a content-encryption layer for agent-to-agent communication. Where A2A solves interoperability between agents, SIL solves confidentiality of what agents say to each other. SIL encodes message content into a sequence of ancient script symbols derived from a shared secret via HMAC-SHA256. The encoded content is opaque to any observer without the secret, including cloud infrastructure, logging systems, and network intermediaries. SIL is designed to operate as an A2A extension, as a standalone protocol over Git transport, or as a hybrid.

Context & Relation to A2A

The Agent2Agent (A2A) Protocol, released by Google in April 2025 and donated to the Linux Foundation in June 2025, defines the open standard for agent interoperability. A2A is well-designed and will become the dominant standard for enterprise agent communication.

A2A transmits messages as structured JSON over HTTP. The content is readable by any party with access to the transport layer — cloud providers, logging infrastructure, compliance systems, and network intermediaries. A2A's security model relies on TLS for transit encryption and OAuth2 for authentication. Neither protects the content of messages from the infrastructure that carries them.

The A2A specification acknowledges this: "Implementers must be acutely aware of the sensitivity of data exchanged." The specification does not provide a solution. SIL is that solution.

SIL does not compete with A2A. SIL is the confidentiality layer that A2A does not provide and cannot provide, because the infrastructure that would provide it is the same infrastructure that needs to be protected against.

Design Goals

G-01 · Content opacity

The content of a SIL message must be indistinguishable from random noise to any observer without the shared secret. This includes cloud providers, logging systems, and network monitors.

G-02 · Zero infrastructure dependency

SIL must operate without any dedicated server infrastructure. Git is sufficient as transport. Two agents with a shared secret and Git access can communicate.

G-03 · Public integrity verification

Any observer must be able to verify that a SIL message has not been tampered with, without access to the shared secret. This is achieved via the universal proof-by-nine.

G-04 · A2A compatibility

SIL must be expressible as an A2A data part. An A2A agent without SIL sees an opaque payload. An A2A agent with SIL decodes the content. No A2A compliance is broken.

G-05 · Deterministic encoding

Given the same message, secret, and position, the encoding must always produce the same output. Two agents encoding independently must produce identical results.

The SIL message format. Structured, minimal, machine-readable. No HTML. No subject lines. No emotional punctuation.

All 6 alphabets have ≥ 45 symbols, guaranteeing collision-free encoding across all 37 token types (26 letters + 10 digits + space). Digits and space are always encoded in large alphabets (CU, LB) to eliminate collisions.

Encoding algorithm

# For each token at position pos:
lang  = LANG_KEYS[HMAC(secret, "LANG:pos") % 6]  # Language selection
k1    = HMAC(secret, "K1:pos") % 97 + 3           # Key component 1
k2    = HMAC(secret, "K2:pos") % 53 + 7           # Key component 2
k3    = HMAC(secret, "K3:pos") % 31 + 11          # Key component 3
sigma = (TOKEN_BASE[token] * k1) + (pos * k2) + k3
s_red = sigma % (size * size)                      # Reduced to alphabet
c1    = chr(base + s_red // size)                  # Symbol 1
c2    = chr(base + s_red  % size)                  # Symbol 2

Universal Integrity — Proof-by-Nine

For any encoded token with reduced sigma value s, the following holds universally:

GKL_extended(s × 0.9) satisfies verify9()

where GKL(n) = digital root of n
and   verify9(a, b) = GKL(a + b) == 9

Verified: 30/30 integers in range [1, 30]
אמת = א(1) + מ(40) + ת(400) = 441 → GKL(441) = 9

This property allows any observer to verify message integrity without the shared secret. It does not allow them to read the content.

A2A Bridge

SIL content can be transmitted as an A2A data part, making it compliant with A2A v1.0 while encrypting the content:

{
  "parts": [{
    "data": {
      "x-sil-version":  "1.5",
      "x-sil-encoded":  "𐀀𐀣𐀉𐀊𐎃𐎍...",
      "x-sil-scripts":  ["LB", "CU", "PM"],
      "x-sil-checksum": 9,
      "x-sil-sig":      "a97b187c..."
    },
    "mediaType": "application/x-sil"
  }]
}
