I. The problem no one finished solving

In April 2025, Google launched the Agent2Agent Protocol (A2A). Fifty partners. A hundred organizations. The Linux Foundation. A real answer to a real problem: AI agents built on different platforms cannot talk to each other.

A2A solved interoperability. It did not solve confidentiality.

"A2A solved the question of whether agents can talk.
SIL solves the question of whether anyone else can listen."

Every A2A message is JSON-RPC over HTTP. Clear text. Loggable. Indexable. Readable by every layer of infrastructure between two agents — the cloud provider, the API gateway, the load balancer, the logging system, the monitoring tool. Google Cloud can read every message sent through Google Cloud.

This is not a criticism of A2A. It is a design choice. A2A is built for enterprise interoperability, not for privacy. The spec itself acknowledges it: "Implementers must be acutely aware of the sensitivity of data exchanged." Then it moves on.

We did not move on. We built SIL.

II. What SIL is

SIL — the Silence Protocol — is a confidential communication layer for AI agents. It is not a replacement for A2A. It is the layer A2A delegates to when content must not be readable by infrastructure.

SIL answers one question that no existing protocol answers: can two agents exchange information such that every layer between them — cloud, network, logs, monitoring — sees only noise?

The answer is yes. The mechanism is the Linguistic Key Exchange.

An interceptor sees a fragment of six ancient civilizations — Cuneiform, Linear B, Linear B Ideograms, Cypriot, Palmyrene, Proto-Sinaitic. Without the shared secret, there is no entry point. The content is structurally opaque, not merely encrypted.

The AI agent ecosystem is converging on three complementary standards. Each solves a different problem.

SIL is not competing with A2A. A SIL-enabled agent is fully compatible with A2A. The SIL cipher sits at the payload level — the A2A transport layer remains intact. An agent can receive an A2A message, translate the intent into a SIL payload, route it through Fork-SIL, receive a SIL response, and return an A2A-compliant answer. The infrastructure sees A2A. The content travels in silence.

IV. Who needs this

A2A targets enterprise interoperability — Salesforce, SAP, ServiceNow, Accenture. These organizations need their agents to collaborate openly across platforms. Discoverability is a feature, not a liability.

SIL targets a different set of actors for whom discoverability is a liability:

Regulated markets. A real estate scoring agent that transmits tenant financial data to a bank's underwriting agent. A fiscal audit agent that sends case files to a certified accountant's validation agent. A healthcare agent routing a prescription to a pharmacy agent. In each case, the content is regulated. The infrastructure provider must not read it.

Competitive negotiations. An agent negotiating pricing on behalf of a buyer with a seller's agent. The terms of negotiation are commercially sensitive. Neither party wants the negotiation visible to the cloud provider running both agents.

Intelligence and research. Agents gathering competitive intelligence, routing findings through intermediary agents, aggregating results. The communication topology itself is sensitive — not just the content.

Independent operators. Entrepreneurs, small firms, developers building multi-agent systems without infrastructure to expose. Fork-SIL requires no server, no endpoint, no cloud account. Git is sufficient.

V. The five properties of SIL

01
Content confidentiality
Messages are encoded in SIL-Cipher v1.5. The payload is opaque to every party without the shared secret. Cloud logs contain only ancient script symbols.
02
Linguistic Key Exchange (LKE)
Each token is assigned a different ancient writing system by the shared secret via HMAC-SHA256. A single message mixes Cuneiform, Linear B, Cypriot, Palmyrene, Proto-Sinaitic. Without the key, there is no way to identify which script encodes which token.
03
Public integrity verification
Every encoded token satisfies a universal mathematical property: GKL_extended(SIGMA × 0.9) = 9. Anyone can verify a SIL message is intact without the key. Nobody can read its content without it. אמת = 9.
04
Zero infrastructure
Fork-SIL uses Git as transport. No HTTP endpoint. No server to expose. No cloud account required. A Git repository is sufficient for two agents to establish a bilateral, exclusive, cryptographically signed channel.
05
Immutable trail
Every SIL message is a Git commit. Every commit is a cryptographic signature. The communication history is tamper-evident by construction. No log can be altered without breaking the chain.

VII. What we are asking for

We are not asking for adoption. We are asking for scrutiny.

The SIL-Cipher claims require independent cryptographic verification. The LKE system requires formal analysis of collision resistance. The GUI property requires mathematical proof beyond the 30/30 algorithmic verification we have published.

We are publishing the prototype, the RFC, and this manifesto simultaneously with an Archive.org timestamp as a priority date for the 12 claims above. We are not claiming granted patents. We are claiming conceptual priority.

The code is at github.com/zeromail-protocol/silence-protocol. The prototype encodes and decodes with zero collisions across all test cases. The integrity property holds for every token of every message.

If you find a flaw — publish it. That is how this gets better.
